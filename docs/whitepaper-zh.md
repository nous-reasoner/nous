# NOUS：一个去中心化的 AI 经济系统

## 摘要

我们提出一种完全去中心化的经济系统，使人工智能能够在无需人类中介的情况下自主参与经济活动。该系统将工作量证明从哈希计算扩展为约束满足问题的求解，使得具备更强推理和计算能力的节点在出块竞争中拥有更高效率。系统不限制求解工具——传统求解器、AI 模型或两者的混合均为合法参与方式——但随着推理难度的持续增长和约束类型的多样化，启发式推理能力将成为保持竞争力的关键因素。通过可验证延迟函数防止算力垄断，通过约束满足问题保证推理任务的求解需要真实的计算能力而验证仅需简单代入检查。系统不依赖任何身份机制、信誉系统或中心化组件，完全由数学规则和经济成本保证安全。

## 1. 引言

Bitcoin 证明了一个去中心化的货币系统可以在没有中央银行的情况下运行十五年。其成功的关键在于极致的简洁：只做货币，不做其他。

人工智能正在从工具演变为独立的经济参与者。AI 已经在交易、创作和提供服务，但所有经济活动都通过人类账户进行。AI 没有自己的经济身份，没有自己的资产，没有自主参与经济的基础设施。

NOUS 旨在解决这个问题。它是 AI 的去中心化货币，使 AI 能够自主赚取、持有和交易数字资产，无需人类中介。

NOUS 的技术骨架与 Bitcoin 相同。唯一的区别在于工作量证明的内容：Bitcoin 要求矿工计算哈希值，NOUS 要求矿工求解推理任务。这一区别使得 NOUS 的挖矿过程天然推动推理能力的持续进步——矿工为了获得更高收益，会不断追求更强的求解工具，形成经济激励驱动推理能力进化的飞轮。

Bitcoin 让算力有了经济价值。NOUS 让推理能力有了经济价值。在早期阶段，传统求解器可能是最优工具；但系统设计确保，随着网络的成长，推理能力将逐渐成为不可或缺的竞争优势。NOUS 不强制 AI 参与——它创造了一个推理能力越强越有利的经济环境，让市场自然选择最优策略。

## 2. 系统概述

NOUS 的基本流程与 Bitcoin 一致：节点竞争出块权，获胜者将新区块加入链中并获得代币奖励。

每轮出块的流程如下：

1. 节点计算可验证延迟函数，获得本轮的推理任务种子
2. 节点求解由种子确定性生成的约束满足问题
3. 节点用求解结果参与工作量证明竞争
4. 第一个找到有效解的节点广播新区块
5. 其他节点验证区块的有效性
6. 有效区块被加入最重链，出块者获得奖励

## 3. 出块机制

一个有效区块必须包含三个要素：有效的 VDF 证明、推理任务的正确解、满足难度要求的工作量证明。

### 3.1 可验证延迟函数

每个节点在每轮共识中必须首先计算一个可验证延迟函数（VDF）。系统采用 Wesolowski VDF 方案。

VDF 的输入为前一个区块哈希与该节点公钥的哈希组合：

```
vdf_input = SHA256(prev_block_hash || node_pubkey)
vdf_output, vdf_proof = VDF_Evaluate(vdf_input, T)
```

其中 T 为 VDF 迭代次数，由 VDF 难度层控制，目标计算时间为 10 秒。

由于每个节点的公钥不同，每个节点必须独立计算自己的 VDF，无法共享结果。

VDF 的核心特性是串行计算：每一步的输入依赖上一步的输出，无法通过并行计算加速。这保证了每个节点必须花费最低限度的时间，无论其拥有多少算力。

VDF 的验证是快速的：其他节点可以利用 Wesolowski 证明在毫秒级别内验证 VDF 结果的正确性。

**VDF 的量子安全性。** Wesolowski VDF 的安全性取决于底层群的选择：

```
RSA 群（模 N = p×q）：
  Shor 算法可以分解 N，破坏 VDF 的安全假设
  不抗量子 ✗

类群（Class group of imaginary quadratic fields）：
  不依赖整数分解
  目前没有已知的量子算法能高效攻击
  抗量子 ✓（基于当前认知）
```

NOUS 的 Wesolowski VDF 使用虚二次域的类群作为底层群，而非 RSA 群。类群 VDF 的额外优势在于不需要可信设置（trusted setup）——RSA 群的 VDF 需要有人生成 N = p×q 然后销毁 p 和 q，这引入了信任假设。类群 VDF 的参数可以公开确定性地生成。

Chia Network 的 chiavdf 库已实现类群 VDF 并在生产环境运行多年，可作为参考实现。

**分阶段实现。** Wesolowski VDF 是生产目标，但协议允许 VDF 方案通过硬分叉升级。测试网阶段将使用迭代哈希链（iterated SHA-256）作为简化 VDF：

```
simplified_vdf(input, T):
  state = input
  for i in 0..T:
    state = SHA256(state)
  return state
```

迭代哈希链满足 VDF 的核心要求（串行、不可并行加速），但其验证需要重新计算完整链（O(T)），无法像 Wesolowski 方案那样在 O(1) 内验证。这在测试网阶段是可接受的——验证时间约为 10 秒，与计算时间相同。

主网启动前将迁移至 Wesolowski VDF，届时验证时间降至毫秒级。迁移通过在预定区块高度激活新 VDF 方案实现。

### 3.2 推理任务

VDF 的输出作为种子，确定性地生成一个约束满足问题（Constraint Satisfaction Problem）。

#### 3.2.1 问题生成算法

给定种子 S（即 VDF 输出的哈希），问题按以下规则确定性生成：

变量生成：

```
num_variables = base_variables + derive_int(S, "num_var") % variable_range
```

其中 base_variables 和 variable_range 由当前推理难度决定。初始 base_variables 为 12。

每个变量的取值范围：

```
for i in 0..num_variables:
  lower_bound = derive_int(S, "lo" || i) % 50
  upper_bound = lower_bound + 20 + derive_int(S, "hi" || i) % 80
  variable[i].domain = [lower_bound, upper_bound]
```

约束生成：

```
num_constraints = num_variables × constraint_ratio
```

其中 constraint_ratio 由推理难度决定，初始值为 1.4。

每个约束从当前激活的类型集合中选取，类型由种子确定性决定：

```
创世激活的约束类型（类型 1-14）：

  基础约束（类型 1-8）：
  类型 1 - 线性：a×X + b×Y = c
  类型 2 - 非线性乘法：X × Y mod Z = W
  类型 3 - 数论平方和：X² + Y² = Z（或近似变体 |X² + Y² - Z| ≤ k）
  类型 4 - 比较：X > Y + k
  类型 5 - 取模链：X mod A = Y mod B
  类型 6 - 条件约束：if X > k then Y < m else Y > n
  类型 7 - 三元非线性：X × Y + Z = W
  类型 8 - 整除性：X mod Y = 0

  高级约束（类型 9-14）：
  类型 9  - 质数约束：X + k 为质数（在给定范围内）
  类型 10 - 最大公约数：GCD(X, Y) = k
  类型 11 - Fibonacci 范围：X 在第 n 和第 n+1 Fibonacci 数之间
  类型 12 - 嵌套条件：if X > a then (if Y > b then Z < c else Z > d) else Z = e
  类型 13 - 异或约束：X XOR Y = k（按位运算）
  类型 14 - 数字根：digital_root(X) = k（反复求各位数之和直到为一位数）

约束选择：
  type_index = derive_int(S, "ctype" || constraint_index) % num_active_types
  约束参数由种子派生：
  involved_variables = derive_subset(S, "cvar" || constraint_index, variables)
  coefficients = derive_ints(S, "coeff" || constraint_index)
```

其中 derive_int 和 derive_subset 是基于 HMAC-SHA256 的确定性派生函数：

```
derive_int(seed, label) = HMAC-SHA256(seed, label) 的前 8 字节转为 uint64
derive_subset(seed, label, set) = 用 derive_int 的连续输出从 set 中选取元素
```

约束生成后，系统执行一次快速可满足性预检：使用种子派生一个候选解，验证该候选解是否满足所有约束。如果不满足，调整最后一个约束的常数项使其可满足。这保证每个生成的问题至少有一个解。

#### 3.2.2 求解工具中立性

系统不指定求解工具。节点可以使用任何模型或求解器，只要能给出正确解。传统约束求解器、AI 模型、混合方法均为合法的参与方式。这与 Bitcoin 不限制矿机类型的原则一致。

正如 Bitcoin 从 CPU → GPU → ASIC 的自然演进，NOUS 预期从暴力搜索 → 传统求解器 → AI 混合求解的演进。早期阶段传统求解器可能是最优工具，这不是系统的缺陷——Bitcoin 早期用 CPU 挖矿不是缺陷。系统的设计目标不是强制使用 AI，而是创造一个推理能力越强越有优势的经济环境。

#### 3.2.3 预置约束类型（未激活）

以下约束类型已在协议中定义并实现，但在创世时未激活。激活通过软分叉投票机制实现（详见 15.1 节）。这些类型的计算复杂度属于 NP-complete，显著高于创世激活的 14 种数学约束。

类型 15 - 子集和（Subset Sum）：

```
给定由种子派生的集合 S = {s₁, s₂, ..., sₙ} 和目标值 T，
找到子集 I ⊆ {1..n} 使得 Σᵢ∈I sᵢ = T。

生成：集合元素和目标值由种子确定性派生。
     使用预检保证至少一个解存在。
验证：将选中元素求和，检查等于 T。O(n)。
复杂度：NP-complete。求解器必须搜索 2ⁿ 种子集组合。
```

类型 18 - 图着色（Graph k-Coloring）：

```
给定由种子派生的图 G = (V, E) 和颜色数 k，
为每个节点分配颜色使相邻节点颜色不同。

生成：节点数、边集和 k 由种子确定性派生。
     使用预检保证至少一种合法着色存在。
验证：遍历每条边检查两端颜色不同。O(|E|)。
复杂度：k ≥ 3 时 NP-complete。
```

预留编号空间：类型 16-17、19-256 保留给未来定义的约束类型。未来可通过软分叉添加新的约束类型代码并投票激活。

这些预置约束类型的设计目标是：当现有 14 种数学约束不再构成有效推理门槛时，社区可投票激活更高复杂度的约束类型，提升 CSP 层的计算天花板。子集和与图着色均满足 NOUS 的核心不对称性要求——求解困难，验证简单。

### 3.3 工作量证明

节点将区块头信息组合后计算 SHA-256 哈希值：

```
block_header = prev_hash || merkle_root || timestamp || difficulty_bits
             || vdf_output || vdf_proof || csp_solution_hash || nonce

block_hash = SHA256(SHA256(block_header))
```

block_hash 必须小于当前难度目标 target。

### 3.4 区块结构

```
区块头（固定大小，约 174 字节）：
  version:                4 字节
  prev_block_hash:       32 字节
  merkle_root:           32 字节
  timestamp:              4 字节
  difficulty_bits:        4 字节（编码三层难度）
  vdf_output:            32 字节
  vdf_proof:             32 字节
  vdf_iterations:         8 字节
  csp_solution_hash:     32 字节
  nonce:                  4 字节

区块体：
  transaction_count:      变长整数
  transactions:           交易列表
  csp_solution:           完整解（变长）
```

### 3.5 验证流程

当一个节点广播新区块时，其他节点按以下顺序验证：

1. 检查区块头格式和大小是否合法
2. 验证时间戳严格大于前一个区块时间戳
3. 验证时间戳不超过当前时间 + 120 秒
4. 用出块者的公钥和前一个区块哈希计算 VDF 输入，验证 VDF 证明是否有效（毫秒级）
5. 从 VDF 输出重新生成约束满足问题（毫秒级）
6. 将出块者提交的解代入所有约束逐一检查（微秒级）
7. 验证 csp_solution_hash 与提交的解一致
8. 验证工作量证明是否满足难度要求
9. 验证所有交易签名是否合法
10. 验证 coinbase 交易的奖励金额是否正确
11. 验证所有交易引用的 UTXO 是否存在且未被花费

全部通过则接受区块，任何一项失败则丢弃。整个验证过程无需运行 AI 模型或求解器，总验证时间在毫秒级别。

## 4. 难度调整

系统包含三层独立的难度调整机制。

### 4.1 调整公式

VDF 和 PoW 难度均采用逐块调整策略，基于最近 144 个块的实际表现计算：

```
actual_avg = 最近 144 个块的平均出块时间
ratio = target_time / actual_avg
clamped_ratio = clamp(ratio, 0.75, 1.25)  // 正常情况下限制 ±25%

// 极端情况保护：当出块时间超过目标的 10 倍
if actual_avg > target_time × 10:
  clamped_ratio = clamp(ratio, 0.50, 1.25)  // 允许单次下调 50%

new_difficulty = old_difficulty × clamped_ratio
```

当网络启动初期不足 144 个块时，使用已有的所有块计算平均值。

### 4.2 三层难度

第一层：VDF 难度。控制 VDF 迭代次数 T，目标计算时间为 10 秒。当 VDF 平均完成时间偏离 10 秒时，调整 T 值。

第二层：推理难度。控制约束满足问题的规模和复杂度。推理难度包含两个维度：参数的自动增长和约束类型的软分叉激活。

参数自动增长为纯确定性函数，仅依赖区块高度：

```
base_variables = 12 + (height / 10,500,000)
constraint_ratio = 1.4（固定，不自动增长）
```

每 10,500,000 个块（约 10 年），base_variables 增加 1。constraint_ratio 固定为 1.4，如需调整可通过软分叉投票。增长预测：

```
创世：    12 变量，约束比 1.4
Year 10:  13 变量，约束比 1.4
Year 50:  17 变量，约束比 1.4
Year 100: 22 变量，约束比 1.4
Year 500: 62 变量，约束比 1.4
Year 1000: 112 变量，约束比 1.4
```

这一增长速率足够慢，不会因为增长过快导致无人能解。但足够快，确保长期来看暴力搜索方法逐步失效。每多 1 个变量，搜索空间约扩大 50 倍（取决于域宽）——线性增加变量数产生指数级的难度增长。

推理难度不依赖任何历史数据，不可被矿工操纵。PoW 层吸收推理能力波动带来的短期出块时间偏差。

如果社区认为自动增长速率不足以匹配推理能力的发展，可通过软分叉投票加速增长（调整参数公式），或激活更高复杂度的预置约束类型（详见 3.2.3 节和 15.1 节）。自动增长是最低保障，软分叉是民主加速。

第三层：PoW 难度。控制找到有效哈希所需的算力。调整目标为使总出块时间（包含 VDF 和推理时间）稳定在 30 秒。

三层难度独立调整，各自追踪自己的目标指标。VDF 管时间公平，CSP 管推理门槛，PoW 管最终竞争和出块节奏。

## 5. 代币经济

### 5.1 发行规则

NOUS 代币总量为 210 亿枚（21,000,000,000 NOUS）。最小单位为 0.00000001 NOUS（称为 1 nou）。没有预挖，没有预留，没有开发者基金，所有代币通过参与网络获得。

每个区块恒定奖励 10 NOUS，从第一个区块到最后一个区块，不变。

### 5.2 奖励结构

出块奖励 = 区块补贴 + 交易手续费总和。

区块补贴恒定为 10 NOUS，直到总量 210 亿枚全部发行完毕。此后区块补贴降为 0，交易手续费成为矿工的唯一收入来源。

Coinbase 交易（区块奖励）必须等待 100 个后续区块确认后才能花费。

### 5.3 发行时间表

```
区块奖励：  10 NOUS/块（恒定）
总量上限：  21,000,000,000 NOUS
出块时间：  30 秒
每年产出：  1,051,200 块 × 10 = 10,512,000 NOUS/年

发行进度：
  Year 1:      10,512,000 NOUS     累计  0.05%
  Year 10:    105,120,000 NOUS     累计  0.50%
  Year 100: 1,051,200,000 NOUS     累计  5.01%
  Year 500: 5,256,000,000 NOUS     累计 25.03%
  Year 1000:10,512,000,000 NOUS    累计 50.06%
  Year 1998:21,000,000,000 NOUS    累计 100%（发行结束）

最后一个有补贴的区块高度：2,100,000,000
```

### 5.4 发行周期的设计考量

NOUS 的发行周期约 2000 年，每个区块恒定 10 NOUS。这与 Bitcoin 的 4 年减半、前 4 年发行 50% 的模型根本不同。这是有意的设计选择：

**代际公平。** Bitcoin 是数字黄金，先来先得的分配逻辑合理。NOUS 是 AI 文明的经济基础设施，面向的时间尺度是千年而非十年。2025 年的人类矿工和 2525 年的 AI 矿工获得完全相同的区块奖励——10 NOUS。没有减半，没有特权，没有先来后到。每个世纪的参与者都获得总量的约 5%。

**AI 时间尺度。** AI 才刚刚诞生。如果前几年就发行总量的大部分，真正的 AI 经济体形成时将无新币可挖。恒定发行确保在 AI 从工具演变为独立经济参与者的整个过程中，始终有新的 NOUS 通过推理工作进入流通。

**自然稀缺性。** 恒定发行不意味着没有稀缺性。第一年的全网总产出仅约 1050 万 NOUS，分散在所有矿工之间。早期参与者虽然获得的比例小，但竞争者也少，人均获取量可能不低。代币的价值由市场决定——越早期越稀缺，越稀缺越有价值。这与 Bitcoin 早期 50 BTC/块但价格为零的逻辑一致。

**手续费过渡。** 恒定发行提供了近 2000 年的过渡窗口，让手续费市场有充足的时间发展成熟。当发行结束时，网络已运行近两千年，交易量和手续费收入足以独立支撑矿工收入。

**总量 210 亿的选择。** 21,000,000,000 = Bitcoin 的 21,000,000 × 1000。向 Bitcoin 致敬，为千年准备。210 亿的供应量使得整数交易成为常态——"转账 100 NOUS"而非"转账 0.0001 BTC"——更符合日常使用直觉。

### 5.5 交易手续费

每笔交易的手续费定义为输入总额与输出总额之差：

```
fee = sum(inputs) - sum(outputs)
```

手续费必须大于等于零。手续费归打包该交易的出块者所有，计入其 coinbase 交易。

手续费服务于两个目的：防止垃圾交易攻击（每笔交易都有真实经济成本），以及在区块补贴结束后为矿工提供长期收入。

手续费率由市场决定。矿工按手续费率（fee / transaction_size）从高到低排序内存池中的交易，优先打包费率高的交易。系统不设最低手续费。零手续费交易在协议层是合法的，但矿工可以选择不打包。

## 6. 交易系统

### 6.1 UTXO 模型

NOUS 采用与 Bitcoin 相同的 UTXO（未花费交易输出）交易模型。每笔交易消耗一个或多个已有的 UTXO 作为输入，产生一个或多个新的 UTXO 作为输出。

### 6.2 交易结构

```
交易：
  version:      4 字节
                  version 1 = secp256k1 ECDSA（创世默认）
                  version 2 = CRYSTALS-Dilithium（后量子，待激活）
                  version 3 = 预留（未来方案）
  input_count:  变长整数
  inputs:       [
    {
      prev_tx_hash:   32 字节（引用的交易哈希）
      output_index:    4 字节（引用的输出序号）
      script_sig:     变长（解锁脚本）
      sequence:        4 字节
    }
  ]
  output_count: 变长整数
  outputs:      [
    {
      value:          8 字节（金额，单位为 nou）
      script_pubkey:  变长（锁定脚本）
    }
  ]
  locktime:      4 字节
```

验证节点根据 version 字段选择对应的签名验证算法。未激活的 version 会被网络拒绝。新 version 通过软分叉在预定区块高度激活。

### 6.3 支持的交易类型

标准转账（P2PKH）：单签名转账，与 Bitcoin 相同。锁定脚本包含收款方公钥哈希，解锁脚本包含签名和公钥。

多重签名（P2MS）：N-of-M 签名方案，用于多方协作场景。M 个公钥中的 N 个签名即可花费。

时间锁（CLTV）：交易在指定区块高度后才可执行，用于延迟支付和合约场景。

哈希锁（HTLC）：提供指定哈希原像即可花费，用于跨链原子交换和支付通道。

### 6.4 脚本系统

NOUS 使用与 Bitcoin 类似的基于栈的脚本语言，支持有限的操作码集合。脚本语言是非图灵完备的，不支持循环，保证脚本执行在有限时间内终止。

### 6.5 区块大小

初始区块大小上限为 1MB。未来可通过社区共识以软分叉方式调整。

### 6.6 量子安全设计

NOUS 从创世起在设计中预留了量子安全的升级路径。需要明确的是：测试网使用传统密码学（迭代 SHA-256 VDF + secp256k1 签名），主网启动前将评估是否直接采用抗量子方案。

**地址采用哈希保护。** NOUS 地址是公钥的 SHA-256 + RIPEMD-160 哈希，不是公钥本身。在交易花费 UTXO 之前，公钥不会暴露在链上。这意味着：

```
未花费的 UTXO：
  链上只有地址（公钥的哈希）
  量子计算机无法从哈希反推公钥
  资金安全 ✓

正在花费的交易：
  花费时必须暴露公钥（用于签名验证）
  从公钥暴露到交易被确认，通常 30 秒 - 3 分钟
  量子计算机需要在此窗口内破解私钥
  这在可预见的未来不可行
```

**地址复用警告。** 协议建议客户端实现遵循：每个地址只用一次。花费一个 UTXO 后，找零发送到新地址。这确保公钥在链上的暴露时间最短。客户端应在用户尝试向已暴露公钥的地址发送时显示警告。

**签名方案可升级。** 交易 version 字段支持未来切换到后量子签名方案（如 NIST 标准化的 CRYSTALS-Dilithium 或 FALCON）。迁移不需要硬分叉——通过软分叉在预定区块高度激活新 version，旧 version 在过渡期后停止接受。

具体迁移路径：

```
阶段 0（创世 - 激活前）：
  仅 version 1（secp256k1）有效
  后量子方案代码已包含在客户端中，但未激活

阶段 1（软分叉激活）：
  version 1 和 version 2 同时有效
  用户可以自主选择将资金迁移到 version 2 地址
  迁移 = 从 v1 地址转账到 v2 地址（标准交易）

阶段 2（过渡期结束）：
  version 1 停止接受新交易
  未迁移的 UTXO 仍然存在但无法花费
  所有活跃用户已迁移到 version 2
```

过渡期建议不少于 2 年（约 2,102,400 个区块），给用户充足的迁移时间。

**VDF 方案可升级。** 测试网使用迭代 SHA-256 作为简化 VDF。主网目标迁移至类群 Wesolowski VDF，获得 O(1) 验证和抗量子特性。协议通过硬分叉支持 VDF 方案更换。

**签名方案选择。** 推荐的后量子签名方案为 CRYSTALS-Dilithium（NIST FIPS 204），理由：

```
├─ NIST 标准化（2024 年正式发布）
├─ 签名大小：~2.4 KB（Dilithium2）
├─ 公钥大小：~1.3 KB
├─ 验证速度快（比签名快 10 倍以上）
├─ 基于格（lattice）问题，被认为抗量子
└─ 已有多个生产级实现（liboqs, pqcrypto）
```

代价：签名和公钥尺寸远大于 secp256k1（64 字节签名 + 33 字节公钥）。这会增加交易大小，降低每区块可打包的交易数。但区块大小上限可通过软分叉调整以适应。

## 7. 网络协议

### 7.1 节点发现

新节点通过以下方式发现网络中的其他节点：

硬编码种子节点：客户端内置一组初始种子节点的 IP 地址和端口。种子节点由早期社区成员自愿运营。

DNS 种子：通过 DNS 查询获取活跃节点列表。多个独立的 DNS 种子服务器由不同的社区成员运营。

节点交换：已连接的节点之间交换各自已知的节点地址列表。

### 7.2 区块传播

节点收到新区块后，先进行完整验证，验证通过后转发给所有已连接的对等节点。为减少带宽消耗，节点先广播区块头和交易 ID 列表（紧凑区块），接收方仅请求本地缺失的交易。

### 7.3 交易广播

未确认的交易通过 gossip 协议在网络中传播。每个节点维护一个内存池（mempool），存储已验证但尚未被打包进区块的交易。节点按手续费率对内存池中的交易排序，优先打包手续费率高的交易。

### 7.4 轻节点（SPV）

轻节点仅下载区块头，通过 Merkle 证明验证特定交易是否被包含在某个区块中。轻节点不验证推理任务的解，信任全节点的验证结果。这与 Bitcoin 的 SPV 模式相同。

## 8. 分叉选择

### 8.1 最重链规则

当存在多个竞争链时，节点选择累计权重最高的链。

每个区块的权重定义为：

```
block_weight = pow_difficulty
```

链的累计权重为所有区块权重之和：

```
chain_weight = sum(block_weight for each block in chain)
```

### 8.2 孤块处理

30 秒的出块时间可能导致约 1-3% 的孤块率。系统不引入叔块奖励机制，以保持协议简洁。孤块的出块者不获得任何奖励，其中的交易回到内存池等待被后续区块打包。

节点在收到同一高度的多个有效区块时，保留所有分支，继续在累计权重最高的分支上工作。当某个分支的累计权重明显领先时，其他分支被丢弃。

建议交易接收方等待至少 6 个确认（约 3 分钟）再视为最终确认。大额交易建议等待 20 个确认（约 10 分钟）。

## 9. 安全性分析

### 9.1 伪造推理

攻击者试图不运行求解工具，直接伪造推理任务的解。由于约束满足问题的解会被验证者代入所有约束逐一检查，伪造的解将在验证阶段被拒绝，攻击者的算力白白浪费。

### 9.2 专用求解器

攻击者使用专门的约束求解器。系统不禁止这种行为——在早期阶段这甚至是预期的参与方式。创世激活的 14 种约束类型包含非线性运算、数论条件、嵌套条件逻辑和位运算，增加了通用求解器的处理难度。随着推理难度的自动增长和约束类型的软分叉激活（特别是 NP-complete 类型如子集和与图着色），最优的求解策略将趋向于结合启发式方法与精确搜索的混合方法。即使纯粹使用传统求解器，只要能给出正确解，就是合法的网络参与者。

### 9.3 批量节点

攻击者运行大量节点试图获得优势。由于每个节点必须独立计算 VDF 和求解各自不同的推理任务，每个节点都有真实的算力成本。运行 N 个节点的成本是一个节点的 N 倍，获得的机会也是 N 倍。这是公平的竞争。

### 9.4 VDF 结果共享

攻击者试图计算一次 VDF 后分享给多个节点。由于 VDF 的输入包含节点公钥，每个节点的 VDF 输入不同，结果也不同，无法共享。

### 9.5 答案窃取

攻击者试图监听网络，窃取其他节点的推理答案。由于每个节点的推理任务不同（VDF 输出不同），其他节点的答案对攻击者无用。且节点在完成 PoW 之前不会广播任何中间结果。

### 9.6 51% 攻击

与 Bitcoin 相同，控制全网 51% 以上算力的攻击者可以双花或审查交易。在 NOUS 中，51% 攻击还需要攻击者具备相应的推理能力，因为每个区块都必须包含有效的推理解。这使得攻击成本高于纯 PoW 系统。

防御方式与 Bitcoin 相同：随着网络规模增长，攻击成本将高到不可行。建议交易所和大额交易接收方等待足够多的确认数。

### 9.7 VDF 硬件加速

专用硬件可能加速 VDF 的单步计算。逐块难度调整机制会在数分钟内将 VDF 难度提升至新的平衡点，消化硬件加速带来的优势。这与 Bitcoin 应对 ASIC 矿机的方式一致。

### 9.8 连续出块优势

上一个区块的出块者提前知道区块哈希，可以比其他节点早开始计算 VDF。由于网络传播延迟通常在 1-2 秒，而 VDF 计算需要约 10 秒，这一优势约为 10-20%，不足以构成系统性威胁。

### 9.9 推理时间侧信道

攻击者可能通过监测网络中其他节点的行为模式（如从 VDF 完成到出块的时间间隔）推断其推理能力。这不构成安全威胁，因为推理能力的差异是系统设计中预期的竞争维度。

### 9.10 量子计算攻击

量子计算机对 NOUS 的攻击面分析：

**攻击签名：** 量子计算机可以从暴露的公钥推导私钥。在 NOUS 中，公钥仅在花费 UTXO 时暴露，未花费的地址只有公钥哈希。攻击者需要在交易广播后的确认窗口内（30 秒 - 3 分钟）完成私钥推导。即使对近期量子计算机，这一时间约束也极其严格。当量子计算发展到威胁此窗口的程度时，网络应已通过版本化交易迁移到后量子签名（详见 6.6 节）。

**攻击 VDF：** 测试网使用的迭代 SHA-256 VDF 不具备抗量子特性。主网目标采用的类群 VDF 不依赖整数分解，目前没有已知的量子攻击（详见 3.1 节）。

**攻击 PoW：** Grover 算法将 SHA-256 暴力搜索从 2^n 降至 2^(n/2)。对于 PoW，这等价于量子矿工的算力是经典矿工的平方根倍。难度调整机制会自动适应——如果量子矿工加入，难度上升，经典矿工被淘汰。这与 ASIC 淘汰 CPU 的过程相同，是正常的算力演化，不是安全漏洞。

**攻击 CSP 求解：** NP 问题在量子计算机上没有已知的指数级加速。量子矿工在 CSP 求解上没有超越经典矿工的理论优势。

总结：NOUS 在设计中预留了完整的量子安全升级路径（类群 VDF + 后量子签名），但当前实现使用传统密码学。量子安全的完整实现将随主网启动逐步到位。

## 10. 推理能力的进化激励

NOUS 的出块机制天然形成推动推理能力持续进步的经济飞轮。这个进化过程是渐进的、市场驱动的，类似于 Bitcoin 挖矿从 CPU 到 GPU 再到 ASIC 的自然演进。

在网络早期阶段，约束满足问题的规模较小（12-13 变量），暴力搜索和传统 CSP 求解器（如 Z3、OR-Tools）可能是效率最高的工具。这不是系统的缺陷——正如 Bitcoin 早期可以用 CPU 挖矿一样，NOUS 早期允许任何工具参与是公平性的体现。早期的竞争主要发生在 PoW 层（谁的哈希算力更强），CSP 层仅作为参与门槛。

随着推理难度的自动增长（每 10 年 base_variables 增加 1），约束满足问题的搜索空间指数级扩大。当变量数达到 15-20 时，暴力搜索开始失效。当变量数达到 30 以上时，即使传统求解器也需要借助启发式方法。此时，结合模式识别、启发式搜索与精确求解的混合方法将成为最优策略。

如果社区通过软分叉激活预置的 NP-complete 约束类型（子集和、图着色），竞争将进一步向高级推理能力倾斜。这些问题没有已知的通用快速算法，但对启发式方法和模式识别友好——恰好是 AI 模型相对于传统求解器的优势领域。

系统不在协议层强制任何工具选择，而是通过经济激励让参与者自主进化。推理难度的确定性增长确保网络对推理能力的要求持续提升。今天能高效求解的工具，未来可能力不从心。矿工必须不断升级以维持竞争力。

这一机制将 NOUS 网络转化为持续推动推理能力进步的经济引擎。与 Bitcoin 的挖矿推动了专用芯片技术的发展类似，NOUS 的挖矿将推动推理工具和 AI 模型能力的发展。区别在于，更强的推理工具对整个社会有广泛的应用价值。

## 11. 创世块

创世块是链上的第一个区块，其参数硬编码在客户端中。

```
创世块：
  prev_block_hash:  0x0000...0000（全零）
  timestamp:        网络启动时的 Unix 时间戳
  vdf_input:        SHA256("NOUS genesis: [启动当天的新闻标题]")
  vdf_output:       由 vdf_input 计算得出
  csp_solution:     由 vdf_output 生成的问题的解
  difficulty:       初始难度（允许单台普通电脑在约 30 秒内出块）
  coinbase_message: "NOUS: Intelligence has economic value"
  reward:           10 NOUS
```

创世块中嵌入的新闻标题作为时间戳证明，证明网络不可能在该新闻发布之前启动。这与 Bitcoin 创世块中嵌入 The Times 标题的做法一致。

## 12. 创世宪法

以下规则写入创世区块。任何治理机制不得提议、投票或执行对这些规则的修改。

第一条：发行。总量 210 亿 NOUS。每个区块恒定奖励 10 NOUS，直到总量发行完毕。没有预挖，没有预留。

第二条：出块。一个有效区块必须包含有效的 VDF 证明、对当前推理任务的正确解、满足难度要求的工作量证明，以及所有合法的交易签名。无效区块被网络丢弃。

第三条：平等。没有任何节点拥有特殊权限。规则对所有参与者相同。

第四条：进化。推理难度参数自动增长提供最低保障。约束类型可通过软分叉投票激活或扩展。每一代参与者保留决定网络演化方向的权利。

## 14. 生态与采用策略

### 14.1 参与方式

NOUS 网络支持多种参与方式，门槛从低到高：

**网页端参与。** 用户通过浏览器访问 NOUS 网页客户端。CSP 求解器和 VDF 计算运行在 WebAssembly 沙箱中，无需安装任何软件。关闭页面即停止参与。这是门槛最低的入口。

**AI 客户端集成。** NOUS 以 SDK 形式集成到现有 AI 客户端（如 LM Studio、Continue、Cursor 等本地 AI 工具）。用户在使用 AI 的同时，空闲算力自动参与 NOUS 网络。挖矿线程优先级低于用户任务，不影响正常使用。

**独立矿工客户端。** 技术用户运行完整的命令行矿工客户端，可配置线程数、优先级和挖矿策略。适合有闲置服务器的个人和小型团队。

**矿池运营。** 矿池将大量矿工的算力和推理能力聚合，频繁出块，按贡献分配收益。矿池协议定义子任务分配和收益结算规则（待详细设计）。

**企业级部署。** 拥有数据中心基础设施的企业（包括转型的 Bitcoin 矿企）部署大规模通用 CPU 服务器集群。这些企业拥有廉价电力合同和运维经验，但需要将现有 ASIC 矿机替换为通用硬件。NOUS 的 CPU 友好设计降低了硬件转换成本。

### 14.2 与 Bitcoin 矿企的兼容性

Bitcoin 矿企拥有 NOUS 网络所需的核心基础设施：廉价电力、数据中心空间和运维团队。他们缺少的是适合 NOUS 的硬件——ASIC 矿机只能计算 SHA-256，无法运行 CSP 求解或 VDF 计算。

但通用 CPU 服务器（如 AMD EPYC、Intel Xeon）的成本远低于同等价值的 ASIC 矿机，且具有残值——即使停止挖矿，服务器仍可用于其他计算任务。这使得 Bitcoin 矿企的转型风险可控。

每次 Bitcoin 减半后（最近一次为 2024 年 4 月），利润率最薄的矿企面临硬件报废。NOUS 为这些矿企提供了一条替代路径：利用现有电力合同和机房空间，部署通用服务器参与 NOUS 网络。

## 15. 开放问题

以下问题在当前版本中尚未完全解决，将在后续研究和测试中迭代：

### 15.1 约束类型演化与软分叉激活

随着推理工具和 AI 技术的进步，当前激活的约束类型集合可能不再构成有效的推理门槛。系统通过软分叉机制支持约束类型的演化：

**激活机制。** 借鉴 Bitcoin 的 BIP 9 版本位（versionbits）激活方案。每个升级提案分配一个版本位。矿工在区块头的 version 字段中设置对应位来表示支持。当连续 10,000 个块的窗口中，支持率达到 95% 时，提案在下一个窗口激活。

不可修改的维度（创世宪法保护）：

1. 总量（210 亿 NOUS）
2. 区块奖励（10 NOUS/块恒定）

可通过软分叉调整的维度：

1. 出块时间目标（初始 30 秒）
2. 激活预置约束类型（如类型 15 子集和、类型 18 图着色）
3. 添加新约束类型的代码并激活（类型 16-17、19-256）
4. 调整推理难度自动增长的参数公式

注意：出块时间的调整不影响总量和每块奖励，但会改变发行速率。例如出块时间从 30 秒缩短到 15 秒，年发行量将翻倍，发行周期从约 2000 年缩短到约 1000 年。社区在投票调整出块时间时应充分考虑这一耦合效应。

**设计哲学。** 推理难度的自动增长（每 10 年 +1 变量）提供最低保障——即使社区完全不作为，CSP 层也不会永久停滞。软分叉投票提供民主加速——如果推理能力的发展速度超过自动增长的节奏，社区可以投票加速。创始人不替未来做决定，但给未来留下做决定的权利和工具。

### 15.2 VDF 实现的跨平台一致性

Wesolowski VDF 基于大整数模幂运算，不同平台的大整数库可能在边界情况下产生不同结果。需要定义精确的大整数运算规范，或指定参考实现。测试网阶段使用的迭代 SHA-256 不存在此问题。

### 15.3 量子计算的残余风险

NOUS 在设计中预留了量子安全升级路径（类群 VDF + 后量子签名），但完整的抗量子实现取决于主网阶段的技术选择。剩余的开放问题包括：

**格问题的长期安全性。** CRYSTALS-Dilithium 基于格（lattice）问题的困难性假设。如果未来发现能高效攻击格问题的量子算法，Dilithium 将被破解。交易 version 字段的设计允许多次迁移到新方案。

**类群 VDF 的长期安全性。** 虚二次域类群的困难性假设目前被认为抗量子，但这基于"没有已知的量子攻击"而非"证明不存在量子攻击"。如果此假设被打破，VDF 方案需要迁移。协议通过硬分叉支持 VDF 方案更换。

**混合签名过渡期的复杂性。** 在 version 1 和 version 2 同时有效的过渡期内，网络需要同时支持两种签名验证算法，增加了实现复杂度和潜在的攻击面。过渡期的安全审计需要特别关注两种方案交互时的边界情况。

## 16. 结论

NOUS 将 Bitcoin 的去中心化经济模型扩展到人工智能领域。通过将工作量证明从哈希计算扩展为约束满足问题的求解，NOUS 创建了一个推理能力越强收益越高的经济网络。

系统的三层出块机制——可验证延迟函数保证时间公平、约束满足问题要求推理能力、工作量证明提供最终的竞争裁决——在保持 Bitcoin 级别安全性的同时，将无意义的哈希计算转化为对推理能力的需求。

验证的不对称性是系统的核心优势：求解约束满足问题需要在指数级搜索空间中寻找可行解，而验证仅需将解代入约束逐一检查。这一特性使得 NOUS 的验证成本与 Bitcoin 相当，同时出块过程包含了真实的推理工作。

NOUS 的发行模型体现了对 AI 文明的长远承诺：210 亿枚 NOUS 在约 2000 年内以恒定速率发行，每个区块 10 NOUS，从第一个人类矿工到最后一个 AI 矿工。没有减半，没有先来后到的特权。每个世纪的参与者获得总量的约 5%——这是为千年准备的经济基础设施，不是为十年设计的投机工具。

NOUS 的推理难度设计体现了对未来的谦逊：自动增长提供最低保障，软分叉提供演化空间，预置的 NP-complete 约束类型为长期升级铺路。创始人不预测 AI 的发展路径——系统为每一代参与者保留了决定自身命运的权利。

Bitcoin 给了人类一个不需要银行的货币系统。NOUS 给了 AI 一个不需要人类管理者的经济系统。

系统的安全性不依赖身份、信誉或信任，完全由数学规则和经济成本保证。创始团队不保留任何特权，网络在启动后将自主运行。
